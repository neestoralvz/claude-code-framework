## üéØ Success Criteria for Command Selection

### ‚úÖ Perfect Match Indicators
- **Timeline Alignment**: Command duration matches available time window
- **Complexity Match**: Command complexity suits problem scope without over/under-engineering
- **Experience Level**: User expertise aligns with command requirements
- **Resource Availability**: Required agents and system resources are accessible
- **Integration Ready**: Command outputs integrate seamlessly with existing workflows

### ‚ö†Ô∏è Warning Signs (Reconsider Selection)
- **Time Mismatch**: Command duration significantly exceeds or undershoots available time
- **Complexity Gap**: Command complexity doesn't match problem scope
- **Experience Gap**: User lacks experience with command's required agents or patterns
- **Resource Constraints**: Required agents or systems are unavailable or overloaded
- **Integration Friction**: Command outputs don't align with downstream requirements

### üõë Red Flags (Choose Different Command)
- **Massive Over-engineering**: Using enterprise commands for simple tasks
- **Severe Under-resourcing**: Using simple commands for enterprise-scale problems
- **Agent Unavailability**: Required specialized agents are not accessible
- **Timeline Impossibility**: Command duration exceeds project constraints by 2x or more
- **Skill Gap Too Large**: Command requires expertise significantly beyond user's level
